# 产品需求文档：GitHub Stars 3D 拼图球体

## 版本
v2 — 毛玻璃方案回退，聚焦拼图球体核心体验

---

## 1. 问题诊断

当前版本（canvas 纹理毛玻璃卡片）的问题：

| 问题 | 根因 | 严重度 |
|------|------|--------|
| 卡片模糊像马赛克 | Canvas 纹理分辨率 192×72，映射到 3D 球面后透视拉伸+低分辨率 → 文字模糊 | 🔴 严重 |
| 毛玻璃渐变层 | 渐变 + 高光 + 内阴影在低分辨率 canvas 上混叠，进一步降低可读性 | 🟡 中等 |
| 视觉效果差 | 用户预期是拼图/马赛克球体，但看到的是模糊色块 | 🔴 严重 |

## 2. 用户核心需求

用户想要一个**3D 拼图球体**，每个拼图片是一个仓库卡片：

1. **拼图拼接** — 卡片紧密排列在球面上，像拼图/马赛克一样组成球体
2. **清晰可读** — 卡片上的文字清晰，不模糊
3. **旋转可控** — 鼠标拖拽旋转不疯转、右键/空格暂停
4. **简洁干净** — 不要花哨效果（不要hover色块、不要毛玻璃）

**关键洞察**：用户对"毛玻璃"不在乎，在乎的是"拼图效果"+"清晰"。

## 3. 技术方案

### 3.1 放弃 canvas 纹理，改用 R3F + drei `<Text>`

**为什么 canvas 纹理不行**：
- 文字在 canvas 上绘制再贴到 GPU texture → 两次采样 → 天然模糊
- 想要清晰文字必须大幅提高 canvas 分辨率（512+），但 527 张 × 512px canvas = 内存爆炸

**替代方案：CSS3DRenderer** 或 **drei `@react-three/drei` 的 `<Text>` 组件**

`@react-three/drei` 的 `<Text>` 使用 **troika-three-text**，通过 SDF（Signed Distance Field）渲染文字，在任意缩放/旋转下保持清晰。

### 3.2 卡片设计

```
┌─────────────────────────────────────────────────┐
│  owner/repo-name                    ★ 12.3k     │  ← 白色文字，无背景
│  ● TypeScript                                  │  ← 语言颜色圆点 + 语言名
└─────────────────────────────────────────────────┘
```

- **无背景色块**：文字直接贴在球面上 → 拼图感更强
- **或者**：纯色半透明卡片背景（单色，无渐变），文字白色
- 卡片尺寸：沿用 Fibonacci 球面间距 × 94%

### 3.3 交互保留

| 功能 | 实现 |
|------|------|
| 鼠标拖拽旋转 | `Math.pow(dist, 0.35) × 0.45`，上限 0.5 |
| 空格暂停 | `keydown Space → toggle pausedRef` |
| 右键暂停 | `contextmenu → toggle pausedRef` |
| 点击打开仓库 | `window.open(repo.htmlUrl)` |
| 悬浮缩放 | `hovered → scale 1.06`，无额外色块 |

## 4. 性能考量

- 527 张卡片，每张用 drei `<Text>`（troika SDF）→ 每张 2-3 个 Text 组件
- `troika-three-text` 为每个 Text 实例创建独立的 SDF 纹理（小纹理，~64-128px），总共 527×2 ≈ 1000 个小纹理 → 可接受
- 球体旋转使用 `useFrame` lerp → 平滑无卡顿

## 5. 验收标准

- [ ] 打开 star.caodan.io，球体上的每张卡片文字清晰可读
- [ ] 卡片紧密拼接（94% 间距），整体呈现拼图球体效果
- [ ] 鼠标移动到屏幕边缘时球体旋转速度温和（不疯转）
- [ ] 按空格键球体暂停 / 恢复旋转
- [ ] 右键点击球体暂停 / 恢复旋转
- [ ] 点击卡片打开对应 GitHub 仓库
- [ ] 页面无 JS 报错
- [ ] 页面加载不卡顿（FPS 保持 30+）
